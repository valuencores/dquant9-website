🔧 주소 검색 문제 해결 가이드

⚠️ 현재 상황:
주소 검색 버튼을 눌렀지만 팝업이 나타나지 않습니다.

📋 즉시 확인할 사항

1 F12를 눌러 개발자 도구를 엽니다
  • Windows: F12 또는 Ctrl + Shift + I
  • Mac: Cmd + Option + I
2 Console 탭으로 이동합니다

다음 메시지 중 하나가 표시되는지 확인하세요:

🔍 예상 콘솔 메시지

✅ 정상 작동 시

✅ DOM 로드 완료 - 이벤트 리스너 등록 시작
✅ 주소 검색 버튼 발견 - 이벤트 리스너 등록
🔍 Daum Postcode API 상태: ✅ 로드됨
✅ 모든 이벤트 리스너 등록 완료

[버튼 클릭 시]
🏠 주소 검색 버튼 클릭됨
🔍 daum 객체 존재 여부: true
🚀 Daum Postcode 실행 시도...
✅ Daum Postcode open() 호출 완료

❌ 오류 발생 시

케이스 1: API 로드 실패

❌ Daum Postcode API가 로드되지 않았습니다.
또는
🔍 daum 객체 존재 여부: false

해결: 인터넷 연결 확인, 페이지 새로고침 (Ctrl + Shift + R)

케이스 2: 버튼 없음

❌ 주소 검색 버튼을 찾을 수 없습니다! ID: btnSearchAddress

해결: 페이지 완전 새로고침 필요

케이스 3: 클릭 반응 없음

(버튼 클릭 시 아무 메시지도 표시되지 않음)

해결: 이벤트 리스너가 등록되지 않음 - 캐시 삭제 후 새로고침

🛠️ 해결 방법

1 강력한 새로고침
  • Windows: Ctrl + Shift + R
  • Mac: Cmd + Shift + R

이 방법은 캐시를 무시하고 페이지를 다시 로드합니다.

2 브라우저 캐시 완전 삭제
  • Ctrl + Shift + Delete (또는 설정 → 개인정보 및 보안)
  • "캐시된 이미지 및 파일" 체크
  • "데이터 삭제" 클릭
  • 페이지 다시 열기
3 팝업 차단 해제 확인
  • 주소창 오른쪽에 팝업 차단 아이콘이 있는지 확인
  • 있다면 클릭하여 "이 사이트의 팝업 허용" 선택
4 다른 브라우저에서 테스트
  • Chrome이 안 되면 → Edge, Firefox 시도
  • 정상 작동하면 원래 브라우저의 캐시/설정 문제

🚀 테스트 페이지로 확인

아래 테스트 페이지를 열어서 API가 정상 작동하는지 먼저 확인하세요:

🧪 주소 검색 API 테스트

💡 직접 입력 방법 (임시 해결책)

주소 검색이 작동하지 않을 경우:

  1. "주소 검색" 버튼 클릭 → Alert 메시지 표시
  2. Alert에서 "확인" 클릭
  3. 우편번호와 주소 필드가 자동으로 활성화됨
  4. 직접 입력:
    우편번호: 03080
    주소: 서울특별시 종로구 청와대로 1
    상세주소: 101호
