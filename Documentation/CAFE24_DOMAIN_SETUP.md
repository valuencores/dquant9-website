### **카페24 로그인 및 접속:**

1. **카페24 관리자 페이지 접속**
   ```
   https://www.cafe24.com
   ```

2. **로그인** → **나의 서비스 관리**

3. **도메인 관리** 클릭

4. `dquant9.com` 선택

5. **DNS 관리** 또는 **DNS 설정** 클릭

---

### **CNAME 레코드 설정:**

#### **Option 1: www 서브도메인 (권장)**

```
타입(Type):    CNAME
호스트(Host):   www
값(Value):     [Step 1에서 받은 Genspark 배포 도메인]
TTL:          3600 (또는 1시간)
```

**예시:**
```
타입: CNAME
호스트: www
값: 4cd1f08e-a30f-4cc7-a848-d444f382f0a8.genspark.ai
TTL: 3600
```

---

#### **Option 2: 루트 도메인 (@) - A 레코드 사용**

**주의:** CNAME은 루트 도메인(@)에 직접 사용할 수 없습니다!  
루트 도메인에는 **A 레코드**를 사용해야 합니다.

```
타입(Type):    A
호스트(Host):   @ (또는 비워두기)
값(Value):     [Step 1에서 받은 IP 주소]
TTL:          3600
```

**IP 주소 확인 방법:**
- Publish 탭에서 제공
- 또는 Genspark 지원팀에 문의

---

### **기존 CNAME 삭제 (필요 시):**

1. 카페24 DNS 관리에서 기존 CNAME 레코드 확인
2. 잘못된 CNAME 레코드가 있다면 **삭제** 또는 **수정**
3. 새로운 CNAME 레코드 추가

## 🚨 **주의사항**

### ❌ **하지 말아야 할 것:**
1. ❌ CNAME을 루트 도메인(@)에 직접 설정 (작동 안 함!)
2. ❌ 배포하지 않고 CNAME만 변경 (오류 발생!)
3. ❌ TTL을 너무 높게 설정 (변경 시 시간 오래 걸림)

### ✅ **해야 할 것:**
1. ✅ **먼저 Publish 탭에서 배포**
2. ✅ www에는 CNAME 사용
3. ✅ @ (루트)에는 A 레코드 사용
4. ✅ TTL은 3600 (1시간) 권장

## 🔄 **기존 CNAME 변경이 필요한 경우**

### **상황:**
이미 카페24에서 CNAME을 설정했지만, 잘못된 주소로 연결되어 있는 경우

### **해결 방법:**

1. **Publish 탭에서 올바른 배포 URL 확인**
2. **카페24 DNS 관리 접속**
3. **기존 CNAME 레코드 찾기**
4. **수정 또는 삭제 후 재생성:**
   ```
   기존: www → old-wrong-domain.com (삭제)
   새로: www → 4cd1f08e-a30f-4cc7-a848-d444f382f0a8.genspark.ai (추가)
   ```
5. **저장 및 적용**
6. **DNS 전파 대기 (1-6시간)**
