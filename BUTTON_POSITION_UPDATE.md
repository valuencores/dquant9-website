# CASE STUDY & ADMIN 버튼 위치 변경

## ✅ 변경 내용

### 🎯 주요 변경사항
1. **버튼 위치**: 좌측 → **우측**으로 이동
2. **버튼 크기**: 가로 폭 **40% 증가** (100px → 140px)
3. **버튼 정렬**: CASE STUDY가 ADMIN 버튼 **바로 위**에 배치
4. **스크롤 버튼**: 두 버튼 위로 이동

                    ↑

## 🎨 디자인 개선

### 1. 가로 크기 증가
- **이전**: 100px (좁고 답답함)
- **현재**: 140px (여유 있고 시각적으로 안정적)
- **증가율**: +40%

### 2. 위치 이동
- **이전**: 좌측 하단 (left: 30px)
- **현재**: 우측 하단 (right: 30px)
- **이유**: 
  - 스크롤 버튼과 그룹핑
  - 오른손 사용자 접근성 향상
  - 시각적 정리

### 3. 정렬 일관성
- 두 버튼의 가로 크기 **완전히 동일**
- 우측 정렬로 깔끔한 수직 정렬
- 버튼 간 간격 일정 (15px)

## 📱 모바일 최적화

### 모바일 특징
- 버튼 크기: 120px (PC 대비 14% 축소)
- 충분한 터치 영역 확보
- 우측 하단 배치 유지
- 스크롤 버튼과 자연스러운 그룹핑

### 터치 타겟 크기
- CASE STUDY: 120×38px = **4,560px²** ✅
- ADMIN: 120×38px = **4,560px²** ✅
- **권장 최소**: 44×44px = 1,936px² (기준 충족)

## 🎯 사용자 경험 개선

### 1. 시각적 계층
```
상단: 네비게이션 메뉴
중앙: 메인 콘텐츠
하단 우측: 
  ↓
  스크롤 버튼 (보조 기능)
  CASE STUDY (중요)
  ADMIN (관리자 전용)
```

### 2. 접근성 향상
- **오른손 사용자**: 엄지로 쉽게 접근
- **일관된 위치**: 항상 우측 하단
- **충분한 크기**: 잘못 누를 확률 감소

### 3. 시각적 정리
- 모든 플로팅 버튼이 우측에 그룹핑
- 콘텐츠 영역 방해 최소화
- 깔끔한 수직 정렬
